Implementation Notes:

  • The units list should display all units across properties, with filtering options.
  • The property filter should allow users to view units from a specific property.
  • The type filter should allow filtering by unit type (studio, 1 bedroom, etc.).
  • The status filter should allow filtering by occupancy status (occupied, vacant, maintenance).
  • The search field should allow searching by unit number, tenant name, etc.
  • Each unit row should display key information: unit number, property, type, status, rent, size, tenant, and lease end date.
  • Status should be color-coded for easy identification.
  • Clicking on a property name should navigate to the property detail page.
  • Clicking on a tenant name should navigate to the tenant detail page.
  • The actions column should provide quick access to view, edit, and delete operations.
  • The "Add Unit" button should open a form to create a new unit.
  • The table should be sortable by clicking on column headers.
  • Pagination should be implemented for large lists of units.
