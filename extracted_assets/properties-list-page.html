Implementation Notes:

  • The search should filter properties in real-time as the user types.
  • The filter dropdowns should allow filtering by company, property type, and occupancy rate.
  • The view toggle should switch between grid and list views.
  • Each property card should display key information: name, address, company, type, units, occupancy, and monthly revenue.
  • Each property card should have quick actions (View, Edit, Delete).
  • The "Add Property" button should open a form to create a new property.
  • Clicking on "View" should navigate to the property detail page.
  • Property images should be actual photos of the properties when available.
  • The occupancy rate should be calculated based on the number of occupied units divided by total units.
  • Monthly revenue should be the sum of all rent payments for the property.
