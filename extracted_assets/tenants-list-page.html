Implementation Notes:

  • The tenants list should display all tenants with filtering and search capabilities.
  • The property filter should allow users to view tenants from a specific property.
  • The status filter should allow filtering by tenant status (active, former).
  • The sort dropdown should allow sorting by different criteria (name, move-in date, lease end date).
  • The search field should allow searching by tenant name, email, phone, etc.
  • Each tenant card should display key information: name, contact details, property/unit, and lease end date.
  • Status should be indicated with a badge (active or former).
  • Clicking on a property name should navigate to the property detail page.
  • Clicking on a unit identifier should navigate to the unit detail page.
  • The actions at the bottom of each card should provide quick access to view, edit, and delete operations.
  • The "Add Tenant" button should open a form to create a new tenant.
  • Pagination should be implemented for large lists of tenants.
  • Former tenants should be visually distinct from active tenants.
