Implementation Notes:

  • The property detail page should display comprehensive information about a property.
  • The header should feature the property image as a background when available.
  • Statistics cards should display real-time data calculated from the property's units and financial information.
  • Units section should show a preview of units with their occupancy status and tenant information.
  • Maintenance section should display recent and ongoing maintenance requests for the property.
  • Multiple tabs allow users to navigate between different aspects of property management.
  • Action buttons should provide quick access to common tasks like adding units, editing property details, etc.
  • Unit cards should be color-coded by status (occupied, vacant, under maintenance) for easy recognition.
  • Clicking on a tenant name should navigate to their tenant profile.
  • Clicking on a maintenance request should open its detailed view.
